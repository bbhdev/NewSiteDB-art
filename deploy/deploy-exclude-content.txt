# ─────────────────────────────────────────────────────────────────────
# Editor-written content data — TWO-LAYER SYNC SEPARATION
# ─────────────────────────────────────────────────────────────────────
#
# These patterns mirror the gitignored content-data block in
# `.gitignore`. They list every file the editor writes at runtime —
# drawings, layouts, palette, masters, image-workshop output, uploaded
# images, etc.
#
# Why excluded from code deploys: servers A and B can edit content
# independently (planned bidirectional content sync). A code deploy that
# also pushed content would clobber whatever the server's editor most
# recently saved. The separation is mandatory.
#
# Sync responsibility moves to a separate layer (NOT deploy.sh) that
# arbitrates per-page using `_sync` stamps embedded in each page.json —
# newer-stamp wins, whole page is the atomic unit. Not yet built; this
# exclude file is the deploy side of the contract.
#
# One-time exception: `deploy.sh --bootstrap` does NOT load this file,
# so a freshly-seeded server can be brought up to local state in one
# pass. After bootstrap, normal `deploy.sh` runs always include this
# file again.
#
# Keep this list in lockstep with the `.gitignore` content section. If
# new editor-written files appear, add them to both.
# ─────────────────────────────────────────────────────────────────────

# Per-page draft/upload area. Covers both `_drafts/images/` (uploaded

# batches under their own pages). Catches both depth levels.
# This subsumes the previous image-workshop-specific block:
# `content/dev/image-workshop/_drafts/` is caught by the 2-level rule.
# `content/dev/image-workshop/image-workshop.txt` is git-tracked and
# rides the normal code mirror — no special re-include needed.
