# ─────────────────────────────────────────────────────────────────────
# Content-layer filter for deploy.sh — Option B (pages = content).
# ─────────────────────────────────────────────────────────────────────
#
# DESIGN: The entire /content/ tree is content-layer EXCEPT a small,
# explicit allow-list of code-side scaffolding files. The content-sync
# layer (per-page `_sync` stamps in page.json, bidirectional A↔B + UI-
# triggered L↔A/B) owns everything else.
#
# This is one-way L→A/B deploy is the wrong tool for multi-master state.
# Pages can be created on any node; the sync layer is the only mechanism
# that can move new pages between nodes. Deploy.sh stays purely code.
#
# WHY NOT --exclude-from with `!` re-includes: rsync's --exclude-from
# does NOT honor `!` for re-includes (that's gitignore syntax). The
# proper rsync mechanism is --filter='merge FILE' with explicit `+`
# (include) and `-` (exclude) prefixes, evaluated FIRST-MATCH-WINS in
# top-to-bottom order. deploy.sh now uses `--filter='merge ...'` to
# load this file.
#
# ONE-TIME EXCEPTION: `deploy.sh --bootstrap` does NOT load this file,
# so a freshly-seeded server can be brought up to local state in one
# pass. After bootstrap, normal `deploy.sh` always loads it again.
#
# ALLOW-LIST RATIONALE: A few files under /content/ are NOT editor-
# written content — they're routing scaffolding for /dev tool surfaces
# (the editor itself) and Kirby's required error page. Those ride with
# code so the editor and error page exist on every node from day one.
# The list is small, stable, and orthogonal to user content.
# ─────────────────────────────────────────────────────────────────────

# Allow /content/ itself to be traversed by rsync (without this, the
# leaf includes below never match because rsync skips the parent dir).
+ /content/

# /dev tool route records — page existence + blueprint binding for the
# four editor surfaces (/dev root, /dev/page, /dev/draw, /dev/image-workshop).
# These are code-side: they define editor routes, hold no user content.
+ /content/dev/
+ /content/dev/default.txt
+ /content/dev/page/
+ /content/dev/page/page.txt
+ /content/dev/draw/
+ /content/dev/draw/draw.txt
+ /content/dev/image-workshop/
+ /content/dev/image-workshop/image-workshop.txt

# Kirby's required error page record (no editable fields; defines the
# 404 page exists with the error blueprint).
+ /content/error/
+ /content/error/error.txt

# Everything else under /content/ is content-layer — owned by the sync
# layer, never touched by deploy.sh in normal mode.
#
# This single rule covers: per-page directories (page.json + .txt
# sidecars, rects.json + .txt sidecars, instances.json + .txt sidecars,
# lines.json + .txt sidecars, groups.json + .txt sidecars), uploaded
# images and their .txt sidecars at any depth, _drafts/ subtrees,
# /content/_shared/ registries, root-level page.json + colors.json,
# root-class records (medium/narrow/wide default.txt — UUIDs are
# editor-generated identity, content-layer), and image-workshop _drafts.
- /content/**
